# C언어 빌드 과정 및 마크다운 작성법 요약

이 문서는 벼리님의 C언어 학습 내용과 마크다운 정리법을 요약한 가이드입니다.

## 1. C언어 프로그램 실행 과정
1. **소스 코드 작성 (`.c`)**: 텍스트 에디터로 코드를 작성합니다.
2. **컴파일 (Compile)**: 사람이 짠 코드를 기계어(오브젝트 파일, `.o`)로 번역합니다.
3. **링크 (Link)**: 번역된 코드와 외부 라이브러리(표준 함수 등)를 하나로 연결하여 실행 파일(`.exe`, `a.out`)을 만듭니다.
4. **실행 (Execute)**: 완성된 파일을 실행하여 프로그램의 결과를 확인합니다.

## 2. 라이브러리와 헤더 파일
- **헤더 파일 (`.h`)**: 함수에 대한 안내판(선언)입니다. 컴파일러 설치 시 함께 제공됩니다.
- **라이브러리**: 함수의 실제 동작이 기계어로 저장된 몸통입니다. 링커가 시스템 폴더에서 찾아 연결합니다.
- **Clang 명령어**: `clang hello.c -o hello`는 컴파일부터 링크까지 한 번에 처리합니다.

## 3. 마크다운(Markdown) 작성 가이드
- **제목**: `#` 개수로 크기 조절 (최대 6개)
- **강조**: `**굵게**`, `*기울임*`
- **코드 블록**: 백틱(`) 3개 뒤에 언어 이름 명시 (예: ```c)
- **인라인 코드**: 문장 중간에 백틱 1개로 감싸기 (예: `int a;`)
- **리스트**: `-` 또는 `1.` 사용
- **Obsidian 팁**: `- [ ]`로 체크리스트를 만들거나 `> [!info]`와 같은 콜아웃 활용
